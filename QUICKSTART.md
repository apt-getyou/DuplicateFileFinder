# 快速使用指南

## 📦 安装

### 方式 1: 使用预编译版本（推荐）

1. 从发布版本下载 `DuplicateFileFinder.exe`
2. 双击运行即可

### 方式 2: 从源码编译

```bash
# 1. 安装 .NET 8.0 SDK
# https://dotnet.microsoft.com/download/dotnet/8.0

# 2. 编译项目
cd DuplicateFileFinder
./build.sh

# 3. 运行程序
bin/Release/net8.0-windows/DuplicateFileFinder.exe
```

## 🚀 第一次使用

### 步骤 1: 添加扫描路径

点击 **"➕ 添加路径"** 按钮，选择要扫描的文件夹。

**建议：**
- 先扫描小范围（如 `D:\Documents`）
- 避免扫描整个系统盘（C:\）
- 可以添加多个路径

### 步骤 2: 配置过滤规则

#### 基本过滤
- ✅ **包含隐藏文件**: 如需扫描隐藏文件，勾选此项
- ✅ **包含系统文件**: 不建议勾选，除非你清楚在做什么
- **最小大小**: 设置文件大小下限（如 1048576 = 1MB），跳过小文件
- **最大大小**: 设置文件大小上限，跳过超大文件

#### 排除规则

**排除扩展名**（每行一个）：
```
.tmp
.log
.cache
```

**排除目录**（预设了程序员常用规则）：
```
.git
node_modules
venv
.venv
__pycache__
.vs
.vscode
obj
bin
```

**程序员特别说明：**
这些目录中的重复文件是**正常**的，不应该删除！

### 步骤 3: 开始扫描

点击 **"🚀 开始扫描"** 按钮，程序会：
1. 显示扫描进度
2. 实时更新状态信息
3. 扫描完成后显示统计信息

### 步骤 4: 查看结果

扫描完成后，你会看到：

**统计信息：**
- 📊 扫描文件总数
- 🔍 重复文件数量
- 💾 浪费空间
- ⏱️ 扫描用时

**重复文件列表：**
- 每行显示一组重复文件
- 包含扩展名、文件大小、副本数、浪费空间
- 展开可查看所有文件路径

### 步骤 5: 处理重复文件

#### 方法 1: 删除文件
1. 在重复文件组中，勾选要删除的文件
2. 点击 **"🗑️ 删除文件"** 按钮
3. 确认删除
4. 文件将被移至**回收站**（可恢复）

**建议：**
- 保留最新或最重要的文件
- 取消勾选不想删除的文件
- 优先处理浪费空间最大的组

#### 方法 2: 移动文件
1. 勾选要移动的文件
2. 点击 **"📦 移动文件"** 按钮
3. 选择目标目录
4. 确认移动

**适用场景：**
- 归档重复文件到备份目录
- 整理文件结构

#### 方法 3: 导出报告
1. 点击 **"📋 导出报告"** 按钮
2. 选择导出格式：
   - **TXT**: 文本格式，便于阅读
   - **JSON**: 机器可读，便于二次处理
   - **CSV**: 表格格式，便于 Excel 打开
3. 选择保存位置
4. 确认导出

## 💾 保存配置

如果你经常扫描相同的路径，可以保存配置：

1. 配置好扫描路径和过滤规则
2. 点击 **"💾 保存配置"** 按钮
3. 选择保存位置和文件名
4. 下次使用时点击 **"📂 加载配置"** 即可

## 📊 使用示例

### 示例 1: 清理照片库

**目标：** 清理 `D:\Photos` 中的重复照片

**配置：**
- 扫描路径: `D:\Photos`
- 排除扩展名: `.db`, `.xml`
- 最小大小: 524288 (500KB)

**操作：**
1. 开始扫描
2. 按浪费空间排序
3. 保留质量最高的照片
4. 删除其他副本

### 示例 2: 清理下载文件夹

**目标：** 清理 `D:\Downloads` 中的重复文件

**配置：**
- 扫描路径: `D:\Downloads`
- 排除扩展名: `.tmp`, `.part`, `.crdownload`
- 最小大小: 1048576 (1MB)

**操作：**
1. 开始扫描
2. 查看重复文件列表
3. 删除不需要的副本

### 示例 3: 程序员的多项目代码管理

**目标：** 找出项目间的真正重复文件（非依赖库）

**配置：**
- 扫描路径: `D:\Projects`
- 排除目录: 使用默认预设（已包含 `.git`, `node_modules` 等）
- 排除扩展名: `.log`, `.cache`

**操作：**
1. 开始扫描
2. 查看重复文件列表
3. 确认哪些是真正的重复（如文档、资源文件）
4. 删除不必要的副本

**注意：** `node_modules` 等依赖库中的重复是正常的，不要删除！

## ⚠️ 注意事项

### 扫描前
- ✅ 先扫描小范围测试
- ✅ 设置合理的过滤规则
- ❌ 避免扫描整个系统盘（C:\）
- ❌ 不要扫描正在使用的程序目录

### 删除前
- ✅ 仔细检查要删除的文件
- ✅ 确认是否真的不需要
- ✅ 优先删除到回收站
- ❌ 不要盲目全选删除

### 特别提醒
- **代码仓库中的重复是正常的**：不同项目可能有相同的依赖库
- **系统文件不要动**：默认已排除系统目录
- **删除后可恢复**：默认移至回收站，30天内可恢复

## 🐛 常见问题

### Q: 扫描速度很慢怎么办？
A:
1. 设置最小文件大小，跳过小文件
2. 减少扫描路径
3. 添加更多排除目录

### Q: 找不到重复文件？
A:
1. 检查过滤规则是否过于严格
2. 确认是否排除了相关目录
3. 查看扫描日志，确认是否正常完成

### Q: 删除后能恢复吗？
A: 可以！默认移至回收站，30天内可恢复。如需永久删除，请在回收站中再次删除。

### Q: 程序无法启动？
A:
1. 确认已安装 .NET 8.0 Runtime
2. 检查防病毒软件是否阻止
3. 尝试以管理员身份运行
